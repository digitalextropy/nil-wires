Notes
A Horizontal Scroll
  • Preserves column alignment — familiar table structure
  • Better for power users comparing values across rows
  • Minimal layout changes needed from desktop
  • User must scroll horizontally to see all columns
  • Easy to miss off-screen data without a scroll hint
  • Horizontal scrolling is less natural on mobile
B Card Reflow
  • No horizontal scroll needed — all data visible
  • Each record is self-contained and easy to scan
  • Better for quick scanning individual athlete records
  • Takes more vertical space — more scrolling overall
  • Harder to compare values across multiple athletes
  • Requires a separate mobile template / component
